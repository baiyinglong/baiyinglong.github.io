git操作说明

1.git的安装

  • windows下安装
    下载地址
  • mac下安装
    下载Homebrew
    拷贝对应脚本到终端下安装HomeBrew
    然后在终端执行 brew install git命令安装*git
    安装xcode会默认下载git

2.告诉git你是谁

如果不配置不能提交代码

查看配置

创建文件夹
自定义快捷键 living template

3.初始化git仓库

ls -al显示全部内容 会产生一个.git的隐藏文件，用来管理我们的文件

4.git中的三个区

工作流

  • 工作区
    通过git add 添加到暂存区

  • 暂存区
    特点:过渡的作用，避免误操作，保护工作区和历史区，分支处理;
    为了保护工作区和历史区，过渡作用
    通过git commit 添加到历史区

  • 历史区
    查看历史状态

修改时通过git status查看当前状态

5.文件操作

  • 向文件内写入内容

  • 查看文件内容
    cat index.txt
    红色表示未加入暂存区内

  • 添加文件到缓存区中

  • 提交到版本区

提交到版本库中通过git log 进行查看

6.比较

比较工作区和缓存区

比较缓存区和历史区

比较工作区和历史区(版本库)

7.撤销

  • 从缓存区中撤销add内容

  • 撤回文件
    如果工作区有更改 ，我想用暂存区域或者历史区的内容去覆盖掉我们的工作区域
    先查找暂存区 ，查找不到 查找历史区
    用暂存区 替换掉工作区里的内容

  • 合并提交
    如果我们使用git commit 会产生一个新的版本，但是我们此次的提交，还想算作上次的提交

如果进入编辑模式按esc键 输入:wq

8.删除

  • 删除缓存区，但是要保证工作区里内容必须不存在

  • 只删除缓存区

  • 同时删除暂存区和工作区

9.恢复

  • 恢复某个版本文件

  • 从版本库拉回代码 回滚 回到过去
    git log 查看版本号

  • 在回到未来
    通过git reflog 查看未来的版本，使用reset回到未来

  • 快速版本回退 #

10.同步远程仓库

  • github
    注册账号
    新建项目

  • 创建仓库后需要关联远程仓库

  • 添加忽略文件

  • 拉取自己的最新代码
  • 拉取老师的最新的代码,如果冲突需要解决冲突

  • 把自己的作业上传到自己的github仓库(如果组长没有要提交的东西可省略)
